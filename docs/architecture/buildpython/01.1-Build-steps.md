# Build steps

This is the current `buildpython` step catalog. List it directly with:

```bash
python -m buildpython --list-steps
```

Run selected steps by number or name:

```bash
python -m buildpython --run-steps=2,18,19
python -m buildpython --run-steps="Compile,Pytest"
python -m buildpython --with-appimage
python -m buildpython --with-black
```

## Step catalog

| # | Step | Purpose | Notes |
|---:|---|---|---|
| 1 | `Compile` | Compile all Python sources as a fast syntax check. | Runs `compileall` over `src/`. |
| 2 | `Pytest` | Run tests with hardware tests disabled by default. | When `coverage.py` is available, this step captures fresh coverage data for Step 18. |
| 3 | `Ruff` | Run lint checks. | Auto-skips if `ruff` is not installed. |
| 4 | `Import Validation` | Import key entry modules to catch import-time failures. | Fast runtime sanity check for main entrypoints. |
| 5 | `Code Markers` | Scan for TODO/FIXME/HACK/NOTE-style markers. | Writes `code-markers.{json,csv,md}`. |
| 6 | `File Size` | Report large Python modules by size buckets, long import blocks, and flat directory hotspots. | Writes `file-size-analysis.{json,csv,md}`. |
| 7 | `Ruff Format` | Check formatting with Ruff's formatter. | Auto-skips if `ruff` is not installed. |
| 8 | `Pip Check` | Validate installed dependency consistency. | Uses `python -m pip check`. |
| 9 | `Import Scan` | Parse imports across the repo and validate required imports. | Excludes unsupported or optional cases that are intentionally tolerated by the runner. |
| 10 | `Repo Validation` | Check packaging, install, and metadata consistency. | Repo sanity check for a releasable public tree. |
| 11 | `Black` | Check formatting with Black. | Not part of the default selection; add with `--with-black` or run explicitly. |
| 12 | `LOC Check` | Report Python files at or above the line-count threshold. | Informational; writes `loc-check.{json,csv,md}`. |
| 13 | `Type Check` | Run mypy-based type checking. | Auto-skips if `mypy` is not installed. |
| 14 | `AppImage` | Build `keyrgb-x86_64.AppImage`. | Packaging step for release artifacts. |
| 15 | `AppImage Smoke` | Smoke-test the AppImage in a minimal container path. | Intended to catch missing runtime dependencies in the bundle path. |
| 16 | `Code Hygiene` | Track defensive-pattern and API-discipline debt. | Writes `code-hygiene.{json,csv,md}` and participates in debt snapshots. |
| 17 | `Architecture Validation` | Validate configured architecture corpus and pattern rules. | Writes `architecture-validation.{json,csv,md}`. |
| 18 | `Coverage` | Build the coverage debt summary and compare against configured baselines. | Writes `coverage-summary.{json,csv,md}` and can fail on missing capture or regressions. |
| 19 | `Exception Transparency` | Track broad exception debt and silent-failure hotspots. | Writes `exception-transparency.{json,csv,md}` and honors typed `@quality-exception exception-transparency: ...` tags. |

## Operational notes

- Hardware tests remain opt-in via `KEYRGB_HW_TESTS=1`.
- Step `18` expects a fresh pytest coverage capture from Step `2`; if it is missing, the report is written with a waiting status and the step fails.
- Debt and report steps are designed to leave structured outputs in `buildlog/keyrgb/` even when they are report-first rather than fully enforcing.
- `AppImage` and `AppImage Smoke` are normal registry steps, so they can appear in bare `python -m buildpython` runs unless you use a narrower profile or skip them explicitly.
