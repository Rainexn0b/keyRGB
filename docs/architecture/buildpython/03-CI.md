# CI

KeyRGB uses GitHub Actions to run `buildpython` as the main validation entrypoint.

## Test workflow

The main CI workflow runs:

```bash
python -m buildpython --profile=ci
```

The current `ci` profile includes:

- `Compile`
- `Import Validation`
- `Import Scan`
- `Pip Check`
- `Pytest`
- `Coverage`
- `Exception Transparency`
- `Architecture Validation`
- `Repo Validation`

Hardware tests remain disabled by default.

## AppImage workflow

CI also runs a packaging job that extends the same profile with AppImage steps:

```bash
python -m buildpython --profile=ci --with-appimage
```

That appends:

- `AppImage`
- `AppImage Smoke`

The resulting AppImage is uploaded as a workflow artifact.

## Release workflow

Tagged releases use the release profile:

```bash
python -m buildpython --profile=release
```

The current `release` profile is the `ci` step set plus `AppImage` and `AppImage Smoke`, matching the packaging path used by the release workflow.

## Practical debugging

When CI fails, reproduce locally with the same profile and inspect `buildlog/keyrgb/`, especially:

- the failing `step-*.log`
- `build-summary.md`
- `debt-index.md`
- any structured report tied to the failing debt step
