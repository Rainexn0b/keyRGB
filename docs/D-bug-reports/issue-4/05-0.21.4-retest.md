# Issue 4 — 0.21.4 reporter continuation

GitHub issue: https://github.com/Rainexn0b/keyRGB/issues/4

Date: 2026-04-08

Reporter findings on `0.21.4`:

- `No backdrop` is fixed in the per-key colors interface.
- `Built-in seed` and `Custom image` now appear broken in that same interface.
- The speed probe flow is clearer but still too fast; a longer dwell is needed.
- Hardware-speed separation still does not look acceptable.
- Resetting the keymap then reassigning LEDs can leave one LED mapped to two
  keys.

Current open items:

1. Fix per-key colors backdrop selection for built-in and custom modes.
2. Improve guided speed-probe timing and clarity.
3. Continue investigating the reported `ite8910` hardware-speed range.
4. Fix keymap reassignment semantics so a physical LED does not remain
   duplicated across keys.

Support output note:

- The latest reporter bundle now records `selection_effect_name = hw:spectrum_cycle`
  and `selection_menu_path = Hardware Effects -> Spectrum Cycle`, which means
  the former hardware-vs-software effect ambiguity has been captured.
