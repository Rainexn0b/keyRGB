## 0.19.0 Reference: Memory And Typing

Last updated: 2026-03-29

## Scope

This note records the remaining useful `0.19.0` ideas that are still worth referencing from `main`, with focus on memory usage and typing-related work.

Explicitly excluded here:

- the `0.19.0` `ite8910` backend implementation and its promotion work
- changes already backported in `0.18.5`
- formatting-only diffs

## Summary

The real remaining `0.19.0` value is mostly on the memory/runtime side.

The typing side is much smaller than the release notes suggest: the meaningful tray `Protocol` and hygiene work is already present on `main`, and the leftover `0.19.0` diff does not contain a large unmerged typing refactor.

## Remaining memory-related reference items

### 1. Keep the SVG-first window icon path in view, but only with optional rasterization

`0.19.0` updated `src/gui/utils/window_icon.py` to prefer `assets/logo-keyrgb.svg` and installed SVG icon locations, while keeping the resized image cache.

Why it is still useful:

- it aligns window icon loading with the vector-first asset direction already used elsewhere
- it keeps the existing cache win for repeated Tk window opens
- it reduces reliance on legacy in-repo PNG assets

Important caveat:

- `0.19.0` imported `cairosvg` directly, which is not safe to copy as-is because it turns an optional dependency into a required import path

If this is picked up later, it should be implemented with the current optional-import approach rather than the `0.19.0` direct import.

Reference files:

- `src/gui/utils/window_icon.py`
- `src/tests/test_window_icon_cache_unit.py`

### 2. Preserve the hot-path buffer reuse test coverage from `0.19.0`

The branch still carries focused tests that are useful as a reference even where the underlying optimization is already on `main`.

Most of the runtime memory work itself is already present, but the leftover branch is still useful as a checklist for protecting it:

- reactive buffer reuse: `src/tests/test_reactive_memory_buffers_unit.py`
- software loop buffer reuse: `src/tests/test_software_memory_buffers_unit.py`
- tray icon cache reuse and cache-key separation: `src/tests/test_tray_icon_draw_cache_unit.py`
- cached window icon reuse: `src/tests/test_window_icon_cache_unit.py`

This is primarily regression-value now, not a missing implementation slice.

### 3. Keep the logical matrix split from backend selection on the backlog

`0.19.0` introduced `src/core/effects/matrix_layout.py` and rewired software/reactive helpers to import `NUM_ROWS` and `NUM_COLS` from that backend-agnostic module instead of `src/core/effects/ite_backend.py`.

Why it is useful:

- removes hidden import-time coupling between software-effect buffers and the legacy backend shim
- makes buffer allocation assumptions explicit and stable
- pairs well with the existing memory-buffer reuse work because the hot paths no longer depend on backend singleton state just to learn matrix size

This is not a typing change, but it is a worthwhile supporting cleanup for the memory/runtime area.

Reference files:

- `src/core/effects/matrix_layout.py`
- `src/core/effects/software/_buffers.py`
- `src/core/effects/software/_effects_basic.py`
- `src/core/effects/reactive/_base_maps.py`
- `src/core/effects/reactive/_ripple_helpers.py`
- `src/core/effects/reactive/render.py`

## Typing assessment

### What remains from `0.19.0`

No substantial unmerged typing improvement was found in the remaining `0.19.0` delta.

The concrete typing-related work the branch advertised is effectively already represented on `main` by existing pieces such as:

- tray `Protocol` definitions in `src/tray/protocols.py`
- code-hygiene tracking for `Any`, `hasattr` coupling, and runtime copy hotspots in `buildpython/steps/step_code_hygiene.py`

The branch comparison did not turn up a meaningful extra `safe_attrs`, `Protocol`, or `Any` reduction slice that is still waiting to be carried over.

### Practical conclusion

For typing, the right interpretation is:

- treat `0.19.0` as confirmation that the direction was correct
- do not plan a separate backport from `0.19.0` for typing alone
- if more type cleanup is wanted, scope it as fresh work against current `main`, especially in remaining GUI and tray UI `Any` call paths

## Not recommended to copy directly

- `0.19.0` `src/gui/utils/window_icon.py` as written, because the direct `cairosvg` import would recreate the optional-dependency release problem
- resource/default-layout simplifications from `0.19.0` that hard-code assumptions more aggressively and drop the ISO extra-key coverage
