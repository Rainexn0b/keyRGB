# Z-legacy — Obsolete documentation

**Scope:** Docs that are no longer authoritative. They are kept for historical
context; active work should reference the lanes in `docs/0-governance/README.md`.

## Categories

- `branches/` — branch/release reference docs for old versions.
- `improvement-roadmaps/` — superseded improvement roadmaps and chunk specs.
- `tech-debt/` — archived tech-debt ledgers and quality campaigns.

## Disposition rules

- Move obsolete docs here rather than deleting them.
- File legacy docs under `Z-legacy/<category>/<YYYY-MM-DD>/`.
- Rename files with a `YYYY-MM-DD-` prefix when the obsolescence date matters.
- Update any internal links that still point to the old location.
