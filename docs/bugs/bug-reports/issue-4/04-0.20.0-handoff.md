# Issue 4 — 0.20.0 handoff and main batch closure

GitHub issue: https://github.com/Rainexn0b/keyRGB/issues/4

Date: 2026-04-04 to 2026-04-05

Reporter findings on `0.20.0`:

- The per-key editor backdrop selector behaved inconsistently with the
  keymap calibrator.
- Software effects were now much better, with no visible blinking during test.
- The guided speed probe appeared to select the software `Spectrum Cycle`
  effect instead of the hardware one.

Main batch outcome:

- Layout/catalog and backend-probed calibrator dimensions were corrected.
- The `ite8910` row translation bug was fixed by flipping logical rows to match
  KeyRGB's existing convention.
- One-logical-key-to-many-cell mapping was implemented across profiles,
  calibrator assignment, editor selection, and reactive typing.
- `ite8910` software blinking was addressed with backend-specific per-key
  mode-maintenance policy.
- `ite8910` hardware speed inversion was fixed while preserving legacy inverted
  mapping for older backends.

Remaining open question:

- Whether the `ite8910` hardware speed range still felt too compressed on
  actual devices.
